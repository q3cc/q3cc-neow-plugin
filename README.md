# q3cc-neow-plugin

一个带有“大喵喵”风格文案的游戏插件，目前主要提供账号信息系统与 24 点玩法。

当前版本：`v0.0.8`

## 功能简介

- 个人信息：`/my`
- 帮助菜单：`/nhelp`（兼容 `/neowhelp`）
- 在线检查：`/ping`
- 24 点菜单：`/24g`
- 24 点开局、难度切换、答题、签到
- 好感度、体力、Star币 统一管理

## 指令列表

### 基础指令

- `/nhelp` - 获取帮助菜单
- `/neowhelp` - `/nhelp` 的兼容别名
- `/ping` - 检查插件是否在线
- `/my` - 查看自己的账号信息

### 24 点指令

- `/24g` - 查看 24 点子菜单
- `/24g start` - 开始一局
- `/24g difficulty` - 查看难度菜单
- `/24g difficulty <0-3>` - 设置难度
- `/24g answer <答案>` - 提交答案
- `/24g sign` - 每日签到

说明：
- 普通 / 困难模式下，回答 `y` 或 `n`
- 极限模式下，需要提交完整算式；若无解可回答 `no`

## 难度规则

- `0 练习`：0 体力 / 0 奖励
- `1 普通`：10 体力 / 随机 `1-4 Star币`、`1-4 好感度`
- `2 困难`：20 体力 / 随机 `1-7 Star币`、`1-7 好感度`
- `3 极限`：30 体力 / 随机 `1-15 Star币`、`1-15 好感度`

当题目数字数量大于 4 且作答时间低于 30 秒时，奖励会缩减。答错会扣除部分 Star币。

## 文件结构

- `index.js` - 插件入口与版本日志
- `apps/neow.js` - Yunzai 实际注册的主插件类
- `meow_user_info.js` - 用户数据、好感度、体力、签到、帮助
- `meow_game_24.js` - 24 点玩法逻辑
- `SPEC.md` - 当前规则说明
- `AGENTS.md` - 协作与贡献说明

## 使用说明

1. 将插件放入机器人插件目录
2. 确保入口文件为 `index.js`
3. 重启机器人后查看日志：

```txt
---------^_^---------
neow插件v0.0.8初始化~
```

## 开发

修改后建议执行：

```bash
node --check index.js
node --check meow_user_info.js
node --check meow_game_24.js
```
