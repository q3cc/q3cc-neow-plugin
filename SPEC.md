# meow 插件规范

## 模块划分
- `apps/neow.js`
  - 负责指令入口、消息匹配、菜单展示与 24 点主流程
- `utils/user-data.js`
  - 负责账号数据、好感度、体力恢复、签到、个人信息展示
- `utils/game24.js`
  - 负责 24 点玩法配置、题库读取、奖励计算与对局键生成
- `utils/ml-game.js`
  - 负责密码破译玩法配置、对局状态与奖励计算
- `utils/blocked-words.js`
  - 负责统一管理色情/低俗英文词汇屏蔽
- `utils/dict-selection.js`
  - 负责管理查词搜索结果的临时选择状态
- `utils/farm-game.js`
  - 负责 farm 附加件装载、农田状态、订单生成与持久化
- `utils/wordle-game.js`
  - 负责 Wordle 猜单词玩法配置、词库校验、对局状态与奖励计算
- `utils/wordle-dict.js`
  - 负责有道词典查询、单词释义解析与格式化
- `utils/rank-render.js`
  - 负责 Star 币排行榜图片渲染与预览 HTML 模板
- `utils/boom-game.js`
  - 负责数字炸弹房间状态、倒计时、回合规则与奖池结算

## 账号规则
- `/my` 是唯一的个人信息查询入口。
- `/rank` 用于查看全局 Star 币排行榜，默认显示前 `10` 名并补充当前用户名次。
- `/rank` 排行榜优先展示最近记录到的 QQ 昵称；若昵称缺失则回退显示 `UID`。
- `/rank` 优先发送排行榜图片；若图片渲染失败、发送失败或环境不支持图片，则自动降级为文字榜单。
- 个人信息面板中的主账号标识统一显示 `UID`。
- 旧用户若缺少 `UID`，则按注册时间从 `1` 开始顺序补发。
- QQ 号仅用于内部识别与管理指令，不作为个人信息面板主展示字段。
- 已废弃 `/24g info`，后续不要恢复。
- 体力上限由 `utils/user-data.js` 统一根据好感度计算。
- 新用户默认属于“陌生旅客”，初始体力为 `150/150`。

## 好感度与体力规则
- `0`：`陌生旅客`，`150` 体力上限
- `30`：`初见相识`，`180` 体力上限
- `100`：`如堕潜梦`，`220` 体力上限
- `200`：`普通朋友`，`270` 体力上限
- `1000`：`朋友之上, 恋人可及`，`230` 体力上限
- `5000`：`如果你是深渊，我愿坠入其中`，`350` 体力上限
- `10000`：`生命中最重要的伙伴！`
  - 达到该等级时，好感度直接写死为 `114514`
  - 后续不再继续增加
  - 体力上限为 `500`
- 好感度描述文案统一使用猫娘口吻，并使用“大喵喵”称呼。

## 总菜单规范
- `/nhelp` 显示总菜单，`/neowhelp` 作为兼容别名保留。
- 总菜单固定为：
  - `/nhelp - 获取指令帮助`
  - `/ping - 在线状态检查`
  - `/my - 获取自己的账号信息`
  - `/rank - 查看 Star 币排行榜`
  - `/dict <词语> - 查词/搜索`
  - `/dict <1-5> - 查看上一轮搜索结果详情`
  - `/sign - 每日签到`
  - `/ml - 密码破译`
  - `/farm - 种田`
  - `/wordle - 猜单词`
  - `/boom - 数字炸弹`
  - `/24g - 查看二十四点子命令菜单`

## 查词规范
- `/dict <词语>`、`/查词 <词语>`、`/dict <词语> -s` 与 `/dict <1-5>` 为查词入口。
- 未提供单词时，必须返回用法说明与示例。
- 查询词仅要求非空，支持中文与英文。
- `-s` 仅作为尾部独立参数使用，表示强制进入搜索模式。
- `/dict <1-5>` 仅用于查看上一轮搜索结果中的对应编号详情。
- 默认流程为：先精确查词，若无有效词条详情则自动回退到 suggest 搜索。
- 查询成功后至少展示：
  - 精确查词命中时：单词原文、英/美音标、中文释义、考试分类（如有）、常见词形变化（如有）
  - 搜索模式命中时：查询词、前 `5` 条候选列表，以及后续 `/dict 1` 查看提示
- 搜索结果需按“会话 + 用户”缓存，默认 `5` 分钟过期。
- `/dict <1-5>` 成功查看详情后，需要清空本轮搜索缓存，避免旧结果重复复用。
- 色情或低俗英文词汇必须直接屏蔽，不进入词典查询流程。
- 查询失败、超时或词典未返回有效内容时，提示用户换一个单词重试。

## 种田规范
- 用户输入 `/farm` 时，必须显示农场等级、主线摘要、地块进度、宠物摘要与快捷指令。
- farm 每日任务固定 `3` 条，按自然日刷新，默认自动领奖。
- 种田玩法按 `用户ID` 全局持久化，不区分群聊与私聊。
- 种田不属于短局互斥游戏，可与 `/24g`、`/ml`、`/wordle`、`/boom` 并行存在。
- 首次进入农场时：
  - 默认拥有 `5` 块普通地
  - 发放新手种子包：`白萝卜 x4`、`卷心菜 x2`、`番茄 x2`
- 农场等级与主账号资料分离，经验来源固定为：
  - 收获：`2 * 实际收获单位数`
  - 卖出：`floor(卖出 Star 币 / 10)`
  - 交订单：`10 + requiredQty * 3`
  - 偷菜成功：`8 * 实际偷到单位数`
  - 主线奖励：按章节 reward 定义
- 地块总数固定 `15` 块：`1-5` 普通、`6-10` 黄土地、`11-15` 黑土地；黄土地产量 `x1.5`，黑土地产量 `x2.0` 且成长 `x0.85`。
- `/farm shop`
  - 只显示当前农场等级已解锁的作物
  - 同时展示种子买入价、种子回收价与果实卖出价
- `/farm help`
  - 展示种田完整指令帮助
- `/farm`
  - 总览只展示快捷操作
  - 必须显示每日任务完成数摘要与 `/farm daily` 入口
  - 主线完成/升级奖励提示与总览信息分两次发送
  - 主线章节必须按顺序解锁，只显示已解锁章节
- `/farm buy <作物别名> [数量]`
  - 购买种子，数量默认 `1`
  - 扣除对应 `Star 币`
- `/farm plant <地块号|起始-结束> <作物别名>`
  - 只能在已拥有空地播种
  - 支持连续地块范围，例如 `1-5`
  - 范围播种必须一次性全部合法且种子足够，否则整次失败，不允许部分播种
  - 扣除该作物定义中的播种体力，总消耗 = 单块体力 * 实际播种地块数
- `/farm water <地块号|all>`
  - 每轮作物只能浇水一次
  - 会按作物基础成长时长缩短固定比例
- `/farm harvest <地块号|all>`
  - 收获成熟作物
  - 使用 `actualHarvest = ceil(baseYield * landYieldMultiplier) - yieldStolen`
- `/farm bag`
  - 查看当前种子、作物与宠物粮背包
  - 种子区必须展示回收单价
- `/farm sell seed <作物别名> <数量|all>`
  - 按种子快照买入价的 `50%` 回收成 `Star 币`
  - 不提供农场经验
  - 不计入 `sell_crop_units`
- `/farm sell <作物别名> <数量|all>`
  - 按作物快照卖出价换成 `Star 币`
- `/farm order`
  - 查看当前 `5` 个订单与刷新剩余时间
  - 每个订单默认同时需求 `2-3` 种作物
- `/farm deliver <订单号>`
  - 交付指定订单，获得 `Star 币 + 好感度`
  - 必须一次性交齐订单内所有作物
  - 完成后立即补一个同槽位新订单
- `/farm daily`
  - 查看当天 `3` 条每日任务、当前进度、奖励与剩余刷新时间
  - 每日任务默认覆盖打开农场、买种子、播种、浇水、收获、卖作物、交订单等常用行为
  - 奖励自动发放，可包含 `Star 币` 与农场经验
- `/farm quest`
  - 查看 `新手教程 / 农场扩张 / 守卫家园` 三条一次性主线当前步骤与进度
- `/farm land`
  - 查看 `15` 块地、解锁等级、价格与拥有状态
- `/farm buyplot <地块号>`
  - 购买已解锁未拥有地块
- `/farm visit <UID>` / `/farm steal <UID> <地块号>`
  - `Lv20` 开放
  - 必须先 visit 再 steal
  - 每人每天最多 `5` 次尝试
  - 每块地每轮作物最多成功被偷 `1` 次
  - 地主至少保留 `1` 个可收获单位
- `/farm pet`
  - 查看当前驻守宠物、已拥有宠物与宠物粮
- `/farm pet shop`
  - 查看核心包与外部附加件提供的宠物、宠物粮商店
- `/farm pet buy <petAlias>` / `/farm pet food buy <foodAlias> [数量]`
  - `Lv20` 开放
  - 购买永久宠物或宠物粮
- `/farm pet use <petAlias>` / `/farm pet feed <foodAlias> [数量]`
  - 同一时间只允许 `1` 只宠物驻守
  - 宠物粮只延长当前驻守宠物的 `guardUntil`
  - 看家总时长上限 `48h`
- `/farm addon`
  - 仅管理员可用
  - 查看 farm 附加件、最近热重载时间与被跳过的坏包

## farm 附加件规范
- 内置基础包使用与外部附加件相同的 schema，位于 `resources/farm-core-addon.json`。
- 外部附加件目录固定为 `data/q3cc-neow-plugin/addons/farm/`，扫描所有 `*.json` 文件。
- 当前同时兼容 `schemaVersion: 1` 与 `schemaVersion: 2`。
- schema v1 固定字段：
  - `schemaVersion`
  - `id`
  - `name`
  - `version`
  - `enabled`
  - `priority`
  - `starterGrants`
  - `crops`
  - `orderTemplates`
- schema v2 额外支持：
  - `crops[].unlockLevel`
  - `pets[]`
  - `petFoods[]`
  - `mainQuestChapters[]`
- 订单模板兼容两种写法：
  - schema v1 / v2 都可使用
  - 旧写法：`cropAlias + qtyMin + qtyMax`
  - 新写法：`requirements[]`，每项包含 `cropAlias + qtyMin + qtyMax`
- `mainQuestChapters[].steps[].type` 仅允许：`open_farm`、`buy_seed`、`plant`、`water`、`harvest`、`sell_crop_units`、`deliver_order`、`reach_level`、`buy_plot`、`harvest_on_land`、`collect_crop_kinds`、`buy_pet`、`buy_pet_food`、`feed_pet_hours`、`visit_farm`、`attempt_steal`、`successful_steal`、`accumulate_guard_hours`。
- 附加件只允许声明固定字段，不支持任意脚本执行。
- 冲突与错误处理：
  - 重复 `id`：后加载包跳过
  - 冲突 `alias`：整个包跳过
  - 订单模板引用不存在的 `cropAlias`：仅跳过该模板
  - JSON 或字段校验失败：跳过坏包，不中断已有可用 registry
- 热重载策略：
  - 监听附加件目录变化
  - 使用 `500ms` 防抖
  - 每次重建完整 registry
  - 只有重建成功才替换当前 registry
  - 重建失败时保留上一份可用 registry

## 密码破译规范
- 用户输入 `/ml` 时，显示密码破译菜单。
- 菜单必须展示：
  - 当前体力与上限
  - 单局体力消耗
  - 当前发送方式
  - `/ml start - 开始游戏`
  - `/ml difficulty - 修改难度`
- `/ml mode`
  - 显示密码破译发送方式菜单
- `/ml mode <auto|image|text>`
  - 设置密码破译发送方式
- `/ml start`
  - 开始一局密码破译
  - 按 `群号 + 用户ID` 独立记录游戏
  - 同一用户同一群内不能重复开第二局
- `/ml difficulty`
  - 显示密码破译难度菜单
- `/ml difficulty <0-4>`
  - 设置密码破译难度
- `/ml <四位数字>`
  - 提交本局四位密码
  - 如果当前用户没有自己的进行中游戏，则不回复
- 反馈颜色说明：
  - `🟢` 数字和位置都正确
  - `🟠` 数字存在但位置错误
  - `🔴` 数字不存在
- 难度规范：
  - `0` 简单：无时限、无次数限制，第 `5` 次答错直接爆炸
  - `1` 普通：120 秒，第 `5` 次答错直接爆炸
  - `2` 困难：90 秒，最多 `5` 次，第 `5` 次答错直接爆炸
  - `3` 极限：45 秒，最多 4 次
  - `4` 另类极限：15 秒，无限次数，第 `5` 次答错直接爆炸
- 发送方式规范：
  - `auto`：优先发送图片，失败时降级为文字
  - `image`：优先尝试图片发送，失败时仍降级为文字
  - `text`：直接发送文字，不进行图片渲染
  - 若当前难度为 `4 另类极限`，则无论用户设置为何，都强制直接发送文字
- 成功破译后按当前难度发放 `Star 币` 与好感度奖励
- 失败后结束本局并公布答案，同时按当前难度随机扣除 `Star 币`

## Wordle 规范
- 用户输入 `/wordle` 时，显示 Wordle 菜单。
- 菜单必须展示：
  - 当前体力与上限
  - 单局体力消耗
  - `/wordle start - 开始游戏`
  - `/wordle difficulty - 修改难度`
- `/wordle start`
  - 开始一局 Wordle
  - 按 `群号 + 用户ID` 独立记录游戏
  - 同一用户同一群内不能重复开第二局
- `/wordle difficulty`
  - 显示 Wordle 难度菜单
- `/wordle difficulty <0-3>`
  - 设置 Wordle 难度
- `/wordle <五字母单词>`
  - 提交本局猜测
  - 只要单词存在于合法猜测词库中即可提交，不要求必须属于答案词库
  - 色情或低俗英文词汇必须直接屏蔽，不允许作为题库答案或合法猜测提交
  - 如果当前用户没有自己的进行中游戏，则不回复
- 反馈颜色说明：
  - `🟢` 字母和位置都正确
  - `🟠` 字母存在但位置错误
  - `🔴` 字母不存在
- 难度规范：
  - `0` 简单：8 次机会，无时限，使用高考基础词库
  - `1` 普通：6 次机会，180 秒，使用高考基础词库 + 四级附加词库
  - `2` 困难：5 次机会，120 秒，使用高考基础词库 + 四级附加词库 + 六级附加词库
  - `3` 极限：4 次机会，60 秒，使用原版 Wordle 答案词库
- 成功猜出后按当前难度发放 `Star 币` 与好感度奖励
- 失败后结束本局并公布答案，同时按当前难度随机扣除 `Star 币`
- Wordle 结算时需要展示答案单词的英/美音标与中文释义

## 数字炸弹规范
- 数字炸弹只允许在群聊中游玩，私聊统一提示“数字炸弹至少需要两人，只能在群聊中游玩”。
- 用户输入 `/boom` 时：
  - 若当前群没有房间，则显示数字炸弹菜单
  - 若当前群已有房间，则显示当前房间状态
- 菜单至少展示：
  - 当前数字炸弹难度
  - 当前 Star 币数量
  - `/boom create - 创建房间`
  - `/boom join - 加入房间`
  - `/boom difficulty - 修改难度`
- `/boom create`
  - 兼容 `/boom creat`
  - 创建一间新的数字炸弹房
  - 建房者自动成为房主并加入房间
  - 同一群同一时间只能有一间数字炸弹房
  - 玩家需要至少 `40` 枚 Star 币才能建房
  - 房间创建后若 `30` 分钟内仍未正式开始，则自动取消
- `/boom join`
  - 加入当前群的待开局房间或倒计时房间
  - 玩家需要至少 `40` 枚 Star 币才能加入
  - 房间最多 `8` 人
- `/boom start`
  - 仅房主可用
  - 开启 `15` 秒倒计时
  - 倒计时结束时若人数仍不足 `2`，则自动取消房间
- `/boom cancel`
  - 仅房主可用
  - 仅可在开局前取消房间
- `/boom leave`
  - 开局前为退出房间
  - 若房主离开，则房主自动转交给当前房间内最早加入的剩余玩家
  - 开局后视为主动引爆，立即按踩雷结算
- `/boom difficulty`
  - 显示数字炸弹难度菜单
- `/boom difficulty <0-3>`
  - 设置数字炸弹个人难度
  - 若玩家已经在房间内，本次房间仍按加入时记录的难度结算
- `/boom <数字>`
  - 仅可在自己回合使用
  - 当前群没有进行中的数字炸弹时，返回提示文案
- 难度规范：
  - `0` 简单：名义 `10` 档，实际入池随机 `3-6`
  - `1` 普通：名义 `20` 档，实际入池随机 `5-8`
  - `2` 困难：名义 `30` 档，实际入池随机 `7-11`
  - `3` 极限：名义 `40` 档，实际入池随机 `9-14`
  - 实际扣币始终小于 `15`
- 开局结算规范：
  - 扣币发生在倒计时结束、正式开局前
  - 开局前若玩家 Star 币低于 `40`，则移出房间
  - 若移出后人数不足 `2`，则整房取消，不扣任何人 Star 币
  - 开局时按当前房间玩家顺序随机打乱出手顺序
  - 炸弹固定藏在 `1-100` 之间
  - 初始可猜范围是 `1-100`
  - 猜错后若数字偏小，则更新下界；若数字偏大，则更新上界
  - 后续只能继续猜当前边界之间的整数
- 结算规范：
  - 猜中炸弹的玩家立即 `boom`
  - 本局立刻结束

## 24点指令规范
- `/sign`
  - 每日签到
  - 兼容 `/签到`、`/qd`、`/checkin`
- `/24g start`
  - 开始一局 24 点

  - 先从内置 `5` 个题库中随机选 `1` 个，再从该题库中抽取题目
  - 每个题库固定 `1000` 道题
  - 五个题库的有解/无解比分别为：`0:10`、`3:7`、`5:5`、`7:3`、`10:0`
  - 难度越高，越容易抽中高解率题库
  - 若用户当前仍有 `/ml` 进行中，则不可开启 `/24g`
- `/24g difficulty`
  - 显示难度菜单
- `/24g difficulty <0-3>`
  - 设置难度
- `/24g answer <答案>`
  - 提交答案
  - 如果当前用户没有自己的进行中游戏，则不回复
## 难度规范
- `0` 练习：`0体力/局`，`0奖励`
- `1` 普通：`10体力/局`，随机获得 `1-4 Star币` 与 `1-4 好感度`
- `2` 困难：`20体力/局`，随机获得 `1-7 Star币` 与 `1-7 好感度`
- `3` 极限：`30体力/局`，随机获得 `1-15 Star币` 与 `1-15 好感度`，需输入完整算式
  - 极限模式答题时，仍只校验提交算式的计算结果是否为 `24`

## 并行与互斥规范
- 不同用户的游戏状态必须按“会话 + 用户”隔离，不能互相串局。
- 同一用户不能同时进行 `/24g` 与 `/ml` 两种游戏。
- 同一用户不能同时进行 `/24g`、`/ml`、`/wordle`、`/boom` 多种游戏。
- `/farm` 为长期养成系统，不参与上述短局互斥限制。
- 用户数据与签到数据改为异步落盘，避免同步写文件阻塞消息处理。

## 结算规则
- 回答正确：
  - 按当前难度范围分别随机发放 `Star币` 与好感度奖励
- 24 点反作弊缩奖改为动态公式：
  - 综合题目数字数量、是否要求算式、题库解率与作答时间进行判定
  - 数字越多、题目越偏低解率、要求完整算式时，超快作答会更容易触发缩奖
  - 缩减后单项奖励最低为 `1`
- 回答错误：
  - 扣除固定 `Star币`
    - 练习：`0`
    - 普通：`2`
    - 困难：`3`
    - 极限：`7`
  - 返回错误提示与正确答案
